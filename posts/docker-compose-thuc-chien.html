Docker Compose thực chiến — quản lý multi-container dễ như ăn cháo

TA Tuan Anh · 15 tháng 2, 2026 · ⏱ 8 phút đọc

Bất kỳ dự án web nào cũng cần ít nhất vài thứ chạy song song: ứng dụng, database, cache, message queue… Trước kia, mình phải start từng thứ thủ công, nhớ port, nhớ config. Kể từ khi biết đến Docker Compose, mọi thứ thay đổi hoàn toàn.

Docker Compose là gì?

Docker Compose là công cụ giúp bạn định nghĩa và chạy nhiều container Docker cùng lúc thông qua một file cấu hình YAML duy nhất. Với một lệnh docker compose up, toàn bộ hệ thống của bạn sẽ sẵn sàng.

Cấu trúc file docker-compose.yml

File Compose được chia thành các khối chính: services, networks và volumes. Đây là ví dụ một stack web điển hình: 

services:
  app:
    build: .
    ports:
      - "8080:8080"
    environment:
      DATABASE_URL: postgres://user:pass@db:5432/mydb
      REDIS_URL: redis://cache:6379
    depends_on:
      db:
        condition: service_healthy
      cache:
        condition: service_started
    networks:
      - backend

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    volumes:
      - pg_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user"]
      interval: 5s
      timeout: 5s
      retries: 5
    networks:
      - backend

  cache:
    image: redis:7-alpine
    command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru
    networks:
      - backend

networks:
  backend:
    driver: bridge

volumes:
  pg_data:

Các lệnh thường dùng

Đây là những lệnh mình dùng hàng ngày:

  • docker compose up -d — khởi động tất cả service ở chế độ nền
  • docker compose down — dừng và xóa container (giữ nguyên volume)
  • docker compose down -v — dừng và xóa cả volume (reset data)
  • docker compose logs -f app — xem log realtime của service app
  • docker compose exec db psql -U user mydb — truy cập trực tiếp PostgreSQL
  • docker compose build --no-cache — rebuild image từ đầu

Tách cấu hình dev / staging / prod

Đừng dùng chung một file Compose cho tất cả môi trường. Hãy tận dụng tính năng override file: 

# Cấu trúc khuyến nghị
docker-compose.yml          # base (services, networks, volumes)
docker-compose.dev.yml      # override cho dev (hot reload, debug port…)
docker-compose.prod.yml     # override cho prod (replica, resource limit…)

# Chạy môi trường dev
docker compose -f docker-compose.yml -f docker-compose.dev.yml up

File dev có thể mount source code để hot reload mà không cần rebuild image: 

# docker-compose.dev.yml
services:
  app:
    volumes:
      - .:/app          # mount source code
      - /app/node_modules
    environment:
      NODE_ENV: development
    command: npm run dev

Healthcheck và depends_on

Một lỗi hay gặp của người mới là rely vào depends_on để đảm bảo database ready trước khi app connect. Nhưng depends_on chỉ đảm bảo container đã start, không đảm bảo service đã sẵn sàng.

Giải pháp đúng: kết hợp healthcheck trong service db và condition: service_healthy trong depends_on — như ví dụ ở trên. Cách này đảm bảo app chỉ start sau khi PostgreSQL thực sự nhận được kết nối.

Kết luận

Docker Compose biến việc setup môi trường dev từ "mất cả buổi sáng" thành "một lệnh". Khi onboard thành viên mới, chỉ cần: clone repo, chạy docker compose up, xong. Không cần tài liệu dài dòng về cách cài từng dependency.

File docker-compose.yml của bạn đã được commit vào repo chưa? Đó là một trong những thứ đầu tiên mình commit trong mọi project.
