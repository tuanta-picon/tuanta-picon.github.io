Bắt đầu với Go — ngôn ngữ của sự đơn giản và hiệu năng cao

TA Tuan Anh · 22 tháng 12, 2025 · ⏱ 16 phút đọc

Khi mình lần đầu nghe về Go (hay Golang), phản ứng đầu tiên là: "Thêm một ngôn ngữ nữa à?". Nhưng sau khi dùng thực tế, mình hiểu tại sao nhiều công ty — từ Google, Uber đến Cloudflare — chọn Go cho infrastructure của họ. Bài này là lý do tại sao và cách bắt đầu.

Tại sao Go?

Go được thiết kế với triết lý đơn giản hơn Java/C++ nhưng nhanh hơn Python/Ruby. Những điểm nổi bật:

  • Compile nhanh: project lớn compile trong vài giây, không phải vài phút.
  • Runtime nhanh: near C performance, GC hiệu quả, low latency.
  • Concurrency first-class: goroutines và channels được build sẵn vào ngôn ngữ.
  • Static binary: build ra một file binary, không cần runtime, deploy dễ như copy file.
  • Standard library mạnh: HTTP server, JSON, crypto, testing — đủ dùng mà không cần framework.

Cú pháp cơ bản

Go có cú pháp tối giản. Không có class, không có inheritance, không có exception:

package main

import (
    "fmt"
    "errors"
)

// Struct thay cho class
type User struct {
    ID    int
    Name  string
    Email string
}

// Method gắn với struct
func (u User) String() string {
    return fmt.Sprintf("%s <%s>", u.Name, u.Email)
}

// Multiple return values — cách Go xử lý error
func findUser(id int) (User, error) {
    if id <= 0 {
        return User{}, errors.New("invalid id")
    }
    return User{ID: id, Name: "Tuan", Email: "tuan@example.com"}, nil
}

func main() {
    user, err := findUser(1)
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println(user)  // Tuan 
}

Lưu ý quan trọng: Go không có try/catch. Error được trả về như một giá trị thông thường — điều này buộc bạn phải xử lý error một cách explicit. Nghe có vẻ dài dòng, nhưng code trở nên rất predictable.

Goroutines và Channels

Đây là "killer feature" của Go. Goroutine là lightweight thread — bạn có thể chạy hàng triệu goroutine trên một máy thông thường. Channel là cơ chế giao tiếp an toàn giữa các goroutine. 

package main

import (
    "fmt"
    "sync"
)

func fetchData(id int, wg *sync.WaitGroup, results chan<- string) {
    defer wg.Done()
    // Giả lập fetch data
    results <- fmt.Sprintf("data from source %d", id)
}

func main() {
    var wg sync.WaitGroup
    results := make(chan string, 5)

    // Chạy 5 goroutine đồng thời
    for i := 1; i <= 5; i++ {
        wg.Add(1)
        go fetchData(i, &wg, results)
    }

    // Đóng channel khi tất cả goroutine xong
    go func() {
        wg.Wait()
        close(results)
    }()

    // Đọc kết quả
    for result := range results {
        fmt.Println(result)
    }
}

REST API đơn giản với standard library

Go có HTTP server built-in, không cần Express hay Flask: 

package main

import (
    "encoding/json"
    "log"
    "net/http"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func usersHandler(w http.ResponseWriter, r *http.Request) {
    users := []User{
        {ID: 1, Name: "Tuan Anh"},
        {ID: 2, Name: "Minh Khoa"},
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(users)
}

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /api/users", usersHandler)

    log.Println("Server running on :8080")
    log.Fatal(http.ListenAndServe(":8080", mux))
}

Chạy lên và test:

go run main.go
curl http://localhost:8080/api/users
# [{"id":1,"name":"Tuan Anh"},{"id":2,"name":"Minh Khoa"}]

Những điều cần làm quen

  • Không có generics... à có rồi: Go 1.18+ có generics. Nhưng cộng đồng vẫn ưa interface{} / any cho nhiều trường hợp.
  • Error handling verbose: if err != nil xuất hiện rất nhiều. Go 2 đang cải thiện điều này.
  • Không có OOP truyền thống: kết hợp struct + interface + embedding thay cho class hierarchy.
  • Package management: dùng Go modules (go.mod). Tương tự npm hay pip nhưng đơn giản hơn.

Kết luận

Go là ngôn ngữ được thiết kế để productive trong các hệ thống lớn với team nhiều người. Cú pháp đơn giản đến mức gần như nhàm chán — và đó là ưu điểm. Khi toàn team viết Go theo một kiểu, code dễ đọc, dễ maintain.

Nếu bạn đang làm backend và chưa thử Go, hãy dành một buổi tối. Build một HTTP server nhỏ, feel concurrency với goroutines. Bạn có thể sẽ bị thuyết phục như mình.
