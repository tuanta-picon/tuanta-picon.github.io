10 tính năng TypeScript ít người dùng nhưng cực kỳ hữu ích

TA Tuan Anh · 3 tháng 2, 2026 · ⏱ 10 phút đọc

Phần lớn dev TypeScript chỉ dùng khoảng 20% tính năng ngôn ngữ này. Phần còn lại — những thứ thực sự giúp code type-safe hơn, ngắn gọn hơn, dễ maintain hơn — bị bỏ qua. Bài viết này giới thiệu 10 tính năng mình hay dùng nhất trong số đó.

1. Template Literal Types

TypeScript cho phép bạn xây dựng string type từ các type khác, giống template literal trong runtime nhưng ở tầng type: 

type Direction = 'left' | 'right' | 'top' | 'bottom';
type Padding    = `padding-${Direction}`;
// "padding-left" | "padding-right" | "padding-top" | "padding-bottom"

type EventName  = 'click' | 'focus' | 'blur';
type Handler    = `on${Capitalize<EventName>}`;
// "onClick" | "onFocus" | "onBlur"

2. Const Assertions (as const)

Khi bạn khai báo một object hoặc array bình thường, TypeScript sẽ widening type. Thêm as const để giữ nguyên literal type: 

// Không có as const — type bị widening
const config = { env: 'production', port: 3000 };
// { env: string; port: number }

// Có as const — type chính xác
const config = { env: 'production', port: 3000 } as const;
// { readonly env: "production"; readonly port: 3000 }

3. Operator satisfies (TS 4.9+)

satisfies cho phép bạn kiểm tra type mà không mất inference. Khác với type annotation thông thường — bạn vẫn giữ được type hẹp hơn của từng property: 

const palette = {
  red:    [255, 0, 0],
  green:  '#00ff00',
  blue:   [0, 0, 255],
} satisfies Record<string, string | number[]>;

// Vẫn giữ được type cụ thể!
palette.red.map(x => x * 2);   // OK — TypeScript biết là number[]
palette.green.toUpperCase();    // OK — TypeScript biết là string

4. infer trong Conditional Types

infer cho phép bạn "trích xuất" một phần type trong conditional type: 

// Lấy return type của bất kỳ function nào
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// Lấy element type của array
type ElementType<T> = T extends (infer E)[] ? E : never;
type N = ElementType<number[]>;  // number

// Unwrap Promise
type Awaited<T> = T extends Promise<infer V> ? V : T;
type R = Awaited<Promise<string>>;  // string

5. Mapped Types với Remapping

Bạn có thể transform tất cả key của một type, kể cả đổi tên key bằng as clause: 

type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface User { name: string; age: number; }
type UserGetters = Getters<User>;
// { getName: () => string; getAge: () => number; }

6. Discriminated Unions — pattern xử lý state an toàn

Thay vì dùng optional fields lộn xộn, hãy dùng discriminated union để model state một cách type-safe: 

type RequestState<T> =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error';   error: Error };

function render<T>(state: RequestState<T>) {
  switch (state.status) {
    case 'loading': return '...';
    case 'success': return state.data;  // TypeScript biết data tồn tại
    case 'error':   return state.error.message;
  }
}

7. Branded Types — ngăn nhầm ID

UserId và OrderId đều là string. TypeScript mặc định cho phép gán lẫn nhau. Branded types ngăn điều đó: 

type Branded<T, Brand> = T & { readonly __brand: Brand };
type UserId  = Branded<string, 'UserId'>;
type OrderId = Branded<string, 'OrderId'>;

function getUser(id: UserId) { /* ... */ }

const orderId = '123' as OrderId;
getUser(orderId);  // ❌ Error! Type 'OrderId' is not assignable to 'UserId'

8. using — Resource Management (TS 5.2+)

TypeScript 5.2 hỗ trợ using keyword theo TC39 proposal. Tự động cleanup resource khi ra khỏi scope: 

function createDbConnection() {
  const conn = openConnection();
  return {
    query: conn.query.bind(conn),
    [Symbol.dispose]: () => conn.close(),
  };
}

{
  using db = createDbConnection();
  await db.query('SELECT * FROM users');
}  // db.close() được gọi tự động ở đây

9. noUncheckedIndexedAccess

Bật option này trong tsconfig.json để TypeScript cảnh báo khi bạn access array index mà không kiểm tra undefined: 

// tsconfig.json
{ "compilerOptions": { "noUncheckedIndexedAccess": true } }

const arr = [1, 2, 3];
const first = arr[0];   // type: number | undefined (thay vì number)
first.toFixed(2);       // ❌ Error — phải check undefined trước

10. Const Type Parameters (TS 5.0+)

Thêm const vào generic để giữ nguyên literal type mà không cần as const ở nơi gọi: 

function identity<const T>(value: T): T {
  return value;
}

const x = identity(['a', 'b', 'c']);
// type: readonly ["a", "b", "c"]  ← literal, không bị widening thành string[]
